`p4-ts` wraps the Perforce `p4` CLI with typed TypeScript primitives for inspection, preview, opt-in sync, and service-oriented workflows. Install snippets and imports use the published package name, `p4client-ts`.

```bash
npm install p4client-ts
```

```ts
import { P4Client } from "p4client-ts";

const p4 = new P4Client();
const environment = await p4.getEnvironment();
const workspaces = await p4.listWorkspaces();
const pending = await p4.listPendingChangelists();
const syncPreview = await p4.previewSync({ fileSpec: "//Project/main/..." });

if (syncPreview.totalCount > 0) {
  await p4.sync({ fileSpec: "//Project/main/..." });
}
```

## Scope

In scope:

- Inspect current environment and workspace state
- List relevant workspaces for the local machine
- Inspect pending changelists and opened files
- Preview reconcile operations
- Preview sync operations and apply sync explicitly
- Compose the same operations through an Effect service wrapper

Out of scope:

- Submit, shelve, unshelve, revert, or resolve workflows
- Changelist, client, stream, or server mutation
- Server administration tasks
